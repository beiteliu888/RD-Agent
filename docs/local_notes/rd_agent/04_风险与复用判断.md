# RD-Agent 风险与复用判断

## 安全与隐私风险

| 风险 | 当前证据 | 处理建议 |
| --- | --- | --- |
| LLM API 发送 prompt / code / logs | README 和 installation docs 要求 chat model；requirements 有 `openai`、`litellm`。 | 真实运行前列出发送内容，优先 mock。 |
| Embedding API 发送文本 | docs 要求 embedding query。 | 禁止把 proprietary alpha、私有报告、凭据放入 embedding。 |
| Kaggle token | data_science docs 要求 `kaggle.json`。 | 不在共享环境读取或配置真实 token。 |
| Docker 执行生成代码 | docs 说 most scenarios 使用 Docker 执行代码。 | 使用隔离环境，限制挂载和网络。 |
| 外部数据下载 | README/docs 包含 wget、Kaggle、Qlib 数据路径。 | 先确认数据许可和存储位置。 |
| Report/PDF 解析 | requirements 有 Azure form recognizer 和 document intelligence config。 | 不上传私有报告到云 OCR。 |
| 交易或投资误用 | README legal disclaimer 明确非投资建议。 | 不接入交易 API，不作为投资决策自动化。 |

## 账号、付费、云端、上传或交易风险

| 类别 | 是否涉及 | 说明 |
| --- | --- | --- |
| OpenAI / Azure / DeepSeek / SiliconFlow | 是 | chat 和 embedding provider 可能付费。 |
| Azure CLI / Azure token provider | 可选 | docs 有 Azure login 配置。 |
| Kaggle | 是，特定 scenario | 需要 token、join competition、下载数据。 |
| Docker Hub / image build | 可能 | Docker build/pull/run。 |
| HuggingFace datasets | 可能 | requirements 有 `datasets`，README 链接 traces。 |
| Trading API | 未见官方入口 | 但 quant 输出不可直接接交易系统。 |

## 值得学习的部分

- 【建议】借鉴 `hypothesis -> experiment -> code -> execution -> feedback -> trace` 对象链。
- 【建议】借鉴 scenario 注入式架构：同一个 RD loop 可以服务 quant、data science、finetune 等不同场景。
- 【建议】借鉴 session dump/resume：每个 step 都可记录和恢复。
- 【建议】借鉴 file-based workspace：生成代码和产物有自然审计边界。
- 【建议】借鉴 UI trace viewer 的方向，但需要更强研究审计字段。

## 可以直接复用的部分

【待验证假设】在未运行前，不建议直接复用业务代码。

可能低风险借鉴：

| 部分 | 复用方式 |
| --- | --- |
| 文档结构 | 作为自研系统的学习参考，不直接复制。 |
| loop step naming | propose / exp_gen / coding / running / feedback / record。 |
| experiment ledger | 可结合本项目需求重写。 |
| session checkpoint 思路 | 复用思想，代码需结合自研系统重写。 |

## 需要重写或隔离的部分

| 部分 | 原因 |
| --- | --- |
| 外部 API 调用层 | 需要更强的人工审批、脱敏和发送内容审计。 |
| Quant validation | 需要强制 OOS、rolling、cross-universe、ablation 和反过拟合门禁。 |
| Research memory | 需要明确保存 hypothesis、failure、decision、why-not，而不仅是 task/session。 |
| UI audit | 需要把每次 hypothesis、数据、指标、失败、审批和下一步形成可查询 ledger。 |
| Data readiness | 需要独立 point-in-time、coverage、universe、survivorship 检查。 |

## 不适合当前系统的部分

- 【建议】不要照搬 Kaggle token / external competition workflow 到 alpha research workflow。
- 【建议】不要把 LLM agent 默认拥有执行权限和外部调用权限。
- 【建议】不要把 README benchmark claim 直接当成生产可用结论。
- 【建议】不要让 agent 自己决定上传、提交、交易或大规模跑付费任务。
