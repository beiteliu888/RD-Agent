## 模拟目标

【事实】本文件当前是运行前报告，不是已完成模拟报告。

【建议】本项目真实闭环模拟应分三层推进：

1. 无密钥本地检查：CLI help、port health check、依赖缺口。
2. 本地静态 walkthrough：沿一个 scenario 的代码路径模拟 propose -> experiment -> coding -> running -> feedback -> record，不调用 API。
3. 用户审批后最小真实运行：限制 `step_n` / `loop_n`，使用 sample data 和临时低权限 API key。

| 字段 | 内容 |

| Goal | 验证 RD-Agent CLI 和本地环境边界，不调用 LLM、不启动 Docker、不下载数据。 |
| Hypothesis | 即使不配置密钥，也能通过 help/port-only health check 识别安装状态和最小可运行阻塞。 |

| Controlled variables | 不写 `.env`；不使用 API key；不运行 Docker；不下载 Qlib/Kaggle data；不启动 UI。 |
| Changed variables | 只执行只读或轻量 CLI 检查。 |
| Expected outcome | 得到 CLI 是否可 import、缺少哪些依赖、端口是否占用、当前 OS 是否匹配。 |
| Commands / entrypoints | 候选：`python -m rdagent.app.cli --help`、`rdagent health_check --no-check-env --no-check-docker`。 |
| Data dependency | 无。 |
| External API / cloud dependency | 无。 |
| Secrets required? | no |
| Expected runtime | 1 分钟以内。 |
| Failure modes | Python 版本不兼容；依赖未安装；包未 editable install；macOS 与 Linux-only claim 不匹配。 |
| Stop condition | 一旦触发 API、Docker、数据下载、长任务或密钥读取需求，立即停止。 |
