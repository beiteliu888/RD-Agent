# RD-Agent 学习索引

## 当前结论

- 【事实】RD-Agent 是 Microsoft 开源的 `Research & Development Agent`，仓库使用 MIT License，Python 包名是 `rdagent`，CLI 入口是 `rdagent = rdagent.app.cli:app`。
- 【事实】官方 README 明确写着 `RD-Agent currently only supports Linux`；当前本地环境是 Darwin/macOS，Python 3.12.9，而官方 CI 主要说明 Python 3.10/3.11。
- 【事实】官方 quickstart 的主场景包括 `fin_factor`、`fin_model`、`fin_quant`、`fin_factor_report`、`general_model`、`data_science`、`llm_finetune`、`ui`、`server_ui`、`health_check`。
- 【事实】真实运行通常会涉及 LLM chat API、embedding API、Docker/conda、Qlib 数据、Kaggle、Hugging Face、Azure/OpenAI/DeepSeek/LiteLLM provider 等外部依赖。
- 【事实】本轮尝试了一个关闭 API 和 Docker 的最小命令：`python -m rdagent.app.cli health_check --no-check-env --no-check-docker`。它没有触发外部调用，但在当前未安装依赖环境中失败，报错为 `ModuleNotFoundError: No module named 'fire'`。
- 【推断】RD-Agent 的 CLI 在分发到具体命令前会 import 多个场景入口，因此即使只想做安全端口检查，也需要先安装运行时依赖。
- 【推断】RD-Agent 比 QuantGPT 更像通用 R&D workflow framework；QuantGPT 更像本地 expression 回测器 + LLM/MCP 工具箱。
- 【建议】当前不要直接跑 `rdagent fin_quant`。如果之后要做最小体验，应先写实验台账，并优先考虑 `fin_factor --loop-n 1`，且只在用户批准 API、Docker、Qlib 数据边界后执行。
- 【待验证假设】RD-Agent 的 hypothesis、feedback、memory、failure reuse 是否由代码强制，还没有验证；需要后续架构回读和小规模受控实验。
