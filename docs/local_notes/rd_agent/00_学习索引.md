# RD-Agent 学习索引

## 当前结论

- 【事实】RD-Agent 是 Microsoft 开源的 `Research & Development Agent`，官方定位是面向 data-driven R&D 的 agent framework；它把流程拆成 Research 提出 hypothesis，Development 实现代码，然后通过实验反馈继续迭代。
- 【事实】RD-Agent 不是 quant-specific 项目。Quant 是其中一个重点 scenario，其他 scenario 包括 Data Science / Kaggle、paper-to-model copilot、financial report-to-factor、LLM fine-tuning、UI trace viewer 等。
- 【事实】官方 README 明确写着当前只支持 Linux；本机是 macOS 工作区，因此真实运行 demo 前需要额外判断是否使用 Linux 环境、容器或远程环境。
- 【事实】多数有意义 workflow 需要 LLM chat model 和 embedding model；部分 scenario 还需要 Docker、Qlib 数据、Kaggle token、外部 dataset、Azure/OpenAI/DeepSeek/SiliconFlow 等 provider 配置。
- 【推断】RD-Agent 的学习价值主要不在“拿来即用”，而在它把 hypothesis、experiment、coder、runner、feedback、trace/session 组织成闭环，这比只搜索 expression 的工具更接近 agentic R&D system。
- 【待验证假设】它是否真正做到强制 validation-gated、failure-aware、audit-ready，需要后续运行或静态模拟后验证；目前只确认了代码中存在 feedback decision、trace/session 和日志机制。
- 【建议】下一阶段不要直接跑 `fin_quant`。先做一个无密钥、无 Docker、无数据下载的本地安全模拟，例如检查 CLI 帮助、port health check、依赖缺口和日志路径。
